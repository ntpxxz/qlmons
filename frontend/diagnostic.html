Troubleshooting Guide

Common Issues & Solutions:

❌ "No results from scan"

  1. Check if SQL Server is running: sqlcmd -S 192.168.101.219 -U DATALYZER -P NMB54321 -Q "SELECT 1"
  2. Verify database created: Run schema.sql
  3. Check .env credentials are correct
  4. Restart application: python app.py

❌ "Connection refused"

  1. Verify SQL Server IP: 192.168.101.219
  2. Check SQL Server is running
  3. Verify ODBC Driver 17 installed
  4. Check firewall allows port 1433

❌ "No active sessions"

  1. This is normal if no applications are connected to SQL Server
  2. Create test connection: Connect SQL Server Management Studio to the database
  3. Then scan again to see the session

❌ ".env not found or credentials wrong"

  1. Create .env: copy .env.example .env
  2. Edit with correct credentials: 
    SQL_SERVER=192.168.101.219
SQL_DATABASE=SqlSecurityHUD
SQL_USER=DATALYZER
SQL_PASSWORD=NMB54321
  3. Restart app

Expected Results

✅ Backend Connection: Should return 200 OK

✅ Sessions API: Returns list of active SQL Server sessions

✅ Pending API: Returns [] if no pending hosts (empty array is OK)

✅ Audit API: Returns audit log entries
